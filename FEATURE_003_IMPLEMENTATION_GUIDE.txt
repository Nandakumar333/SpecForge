## CRITICAL IMPLEMENTATION NOTES

### 1. Result[T, E] Pattern
**ALWAYS** use Result for recoverable errors in core/:
\\\python
def generate_one(...) -> Result[Path, str]:
    ...
    if not render_result.ok:
        return Err(f"Failed: {render_result.error}")
    return Ok(output_path)

# Caller
result = generate_one(...)
if result.ok:
    path = result.value
else:
    print(f"Error: {result.error}")
\\\

### 2. No Exceptions for Control Flow
Use Result, not raise/try, in core modules:
\\\python
# ✗ WRONG
try:
    data = json.loads(config_path.read_text())
except json.JSONDecodeError:
    raise ConfigError("Invalid JSON")  # NO

# ✓ CORRECT
try:
    data = json.loads(config_path.read_text())
except json.JSONDecodeError as exc:
    return Err(f"Malformed config: {exc}")
\\\

### 3. File Checksum Strategy
**Generation**:
1. Render with empty checksum placeholder
2. Compute SHA-256 of rendered content
3. Inject checksum into rendered text via regex
4. Write to disk

**Customization Detection**:
1. Determine domain from filename
2. Read project_name from config.json
3. Re-render template (reproducible context)
4. Compute SHA-256 of fresh render
5. Inject expected checksum
6. Read actual file (normalize line endings)
7. Compare hashes: different → customized=True

### 4. Governance File Format
**Header** (lines 1-10): Generated by SpecForge — do not edit above Rules
**Meta section** (lines 12–19): Jinja2 variables, checksum, precedence
**Precedence section** (lines 21–27): Explanation of conflict resolution
**Rules section** (line 29+): User editable zone

### 5. Stack-Specific vs Agnostic
**Agnostic domains**: architecture, security (always {domain}.prompts.md)
**Stack-specific domains**: backend, frontend, database, testing, cicd
- Stack-specific: {domain}.{stack}.prompts.md
- Fallback: {domain}.prompts.md (if stack-specific not found)
