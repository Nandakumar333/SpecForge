# Quickstart — Research & Clarification Engine

## Prerequisites

- SpecForge installed with Features 001-005
- Project initialized (`specforge init`)
- Architecture decomposed (`specforge decompose`)
- Service specs generated (`specforge specify <service>`)

## Basic Usage

### Clarify a Service Spec

```bash
# Interactive mode — answers recorded in spec.md
specforge clarify ledger-service

# Report mode — generates shareable report file
specforge clarify ledger-service --report

# Using feature number instead of slug
specforge clarify 002
```

### Research Technical Unknowns

```bash
# Generate research.md with structured findings
specforge research ledger-service

# Re-run after spec updates (merges with existing)
specforge research ledger-service
```

## Typical Workflow

```
specforge decompose          # Feature 004: define services
specforge specify <service>  # Feature 005: generate spec.md
specforge clarify <service>  # This feature: resolve ambiguities
specforge research <service> # This feature: research unknowns
specforge plan <service>     # Feature 005: generate plan.md (uses research.md)
```

## Key Files

| File | Location | Purpose |
|------|----------|---------|
| spec.md | `.specforge/features/<slug>/spec.md` | Modified by clarify (Clarifications section appended) |
| research.md | `.specforge/features/<slug>/research.md` | Generated by research command |
| clarifications-report.md | `.specforge/features/<slug>/clarifications-report.md` | Generated by clarify --report |
| manifest.json | `.specforge/manifest.json` | Read for service resolution + boundary analysis |

## Architecture-Specific Behavior

- **Microservice**: Research includes communication patterns, container base images, message brokers
- **Monolith**: Research focuses on module-level concerns; no container/service-mesh topics
- **Post-remap**: Clarify generates extra questions about service boundaries and data ownership
